# 测试用例 — Apex Drive Store V2

> **这是什么**：可执行测试用例。编号 `TC-NNN`，关联 REQ/流程/COPY。前置除注明外均从「重置态：GUEST+PARKED+ONLINE，数据已清空」开始。
> 策略见 [test-strategy.md](test-strategy.md)，覆盖见 [coverage-matrix.md](coverage-matrix.md)。

## A. 完整业务流程（E2E）

### TC-001 实物购物车购买（A-01）｜ REQ-007/010/011/012/013/014
1. 首页点 HERO Banner → 进入 phy_car_001 详情（默认曜石黑、数量1）
2. 点「加入购物车」→ 弹 LoginDialog（预填 admin/123456）
3. 点「登录」→ 对话框关闭，自动续作加购，toast `已加入购物车`(COPY-005)
4. SideNav「购物车」→ 1 项，默认勾选
5. 「前往结算」→ 确认订单（地址张先生/上海浦东新区张江中科路1730号，¥129）
6. 「去支付」→ QR 页（¥129 + 二维码占位 + COPY-009）
7. 「模拟扫码成功」→ 写订单、删除已勾选项、跳支付结果
8. 支付结果：`支付成功` / ORDER-P-001 / ¥129 / 已支付
9. 「查看订单」→ 订单中心首条 ORDER-P-001 / 已支付
- **断言**：订单号=ORDER-P-001；购物车该项已删除；金额=¥129。

### TC-002 实物立即购买不变量（A-02）｜ REQ-008
- 前置：LOGGED_IN+PARKED+ONLINE，且购物车已有 N 项（记录行数/数量/勾选）
1. 进入 phy_ele_001，容量改 128G，数量改 2 → 价格 ¥998
2. 「立即购买」→ 确认订单（source=BUY_NOW）→「去支付」→「模拟扫码成功」
3. 订单中心新增一条 ¥998
- **断言**：新订单 ¥998；**购物车行数/数量/勾选与前置完全一致**。

### TC-003 会员开通（B-01）｜ REQ-005/009/013/015/033
1. 会员服务 → mem_001 详情（¥39 + 权益 + 30天 + 绑定特斯拉 Model Y）
2. 「立即开通」→ 确认开通页（标题 `确认开通`(COPY-031)）
3. 「去支付」→「模拟扫码成功」→ ORDER-M-001、membership=ACTIVE
4. 支付结果：`权益已开通`(COPY-029) / ORDER-M-001 / 已开通
5. 回 mem_001：按钮变 `已开通 · 查看订单`(COPY-023)
6. 「我的」会员卡显示 已开通
- **断言**：ORDER-M-001；membership ACTIVE；再次进入开通流不产生第二张订单（幂等）。

### TC-021 订单详情（P13）｜ REQ-034
- 前置：已完成一张实物订单与一张会员订单。
1. 进入订单中心，点击实物订单卡 → 进入 P13。
2. 校验订单号、状态、下单时间、商品清单、SKU、数量、金额、收货信息。
3. 点击 `返回`(COPY-044) → 回订单中心。
4. 点击会员订单卡 → 进入 P13。
5. 校验权益/绑定车辆/有效期，不展示收货地址。
- **断言**：订单详情只读；行车/断网可查看；不存在 orderNo → COPY-036。

### TC-004 展示服务不可购买（C-01）｜ REQ-006
1. 充电服务 → svc_charge_001 详情：仅 `暂未开放`(COPY-006)，无加购/购买入口
2. 强制 `POST /api/cart {productCode:svc_charge_001}` → 返回 code=2003
- **断言**：页面无写入入口；接口返回 2003。

### TC-005 Demo 重置（DEMO-01）｜ REQ-019
- 前置：已有购物车项 + 订单 + membership ACTIVE + 已登录
1. 点 Demo Bar「重置数据」→ 立即执行（无弹窗）
2. 校验后端 cart/checkout/orders/membership 清空；前端 token/session 清空
3. 自动跳首页；Demo Bar 状态恢复 未登录/驻车/在线
4. 走一次实物购买 → 订单号为 ORDER-P-001
- **断言**：全清；状态默认；序号回 001。

### TC-006 搜索（SEARCH-01）｜ REQ-003
1. 首页搜索「磁吸」→ 结果含 phy_car_001 → 点进详情
2. 搜索「不存在的词」→ `未查询到你想要的结果`(COPY-014) + `返回推荐`(COPY-015)
- **断言**：substring 命中；空结果文案与按钮正确。

## B. 门禁与安全

### TC-007 未登录交易拦截 + 续作（AUTH-01）｜ REQ-020
1. GUEST 点「加入购物车」→ 弹 LoginDialog
2. 输错误密码 → `账号或密码不正确`(COPY-003)，对话框保留
3. 输正确密码 → 关闭并续作加购
4. 重复覆盖入口：立即购买 / 开通会员 / 进购物车 / 进订单中心
- **断言**：5 入口均弹窗；错误密码不关闭；正确密码续作；点窗外不关闭。

### TC-008 行车拦截（D-01）｜ REQ-021/027
1. Demo Bar 切「行车」→ 所有交易按钮置灰（alpha 降低）
2. 点置灰按钮 → toast `行车中仅支持浏览，请在驻车后继续操作`(COPY-001)
3. 在确认订单/QR 页切行车 → 主按钮即时置灰；切回驻车 → 恢复
4. 搜索框禁用并显示语音占位，点击 → `语音能力暂未开放`(COPY-004)
- **断言**：置灰、文案、切换恢复、输入禁用均正确。

### TC-009 断网拦截（D-02）｜ REQ-022
1. Demo Bar 切「断网」→ 交易按钮置灰
2. 点置灰按钮 → toast `当前网络不可用，请检查网络后重试`(COPY-002)
3. 仍可只读进入购物车/订单中心
- **断言**：置灰、文案；只读可进入。

### TC-010 拦截优先级（REQ-024）
1. 同时 行车+断网+未登录 → 点交易按钮
- **断言**：只显示行车提示（DRIVING>OFFLINE>GUEST）；toast 不堆叠（REQ-029）。

## C. 接口/集成（Supertest）

### TC-011 加购门禁矩阵｜ API-009
- 无 token → 1001；X-Demo-Drive=DRIVING → 2001；X-Demo-Net=OFFLINE → 2002；DISPLAY_SERVICE → 2003；quantity=6 → 4000；正常 → 0。

### TC-012 数量边界｜ API-010/REQ-031
- quantity=0 或 6 → 4000；=1/5 → 成功；前端边界 +/- 置灰。

### TC-013 订单号序列｜ PRD §10.5
- 连下两实物单 → ORDER-P-001、ORDER-P-002；一会员单 → ORDER-M-001；重置后再下 → ORDER-P-001。

### TC-014 金额计算｜ §6
- phy_ele_001 ×2 → totalCents=99800（显示 ¥998）；多项合计 = Σ 单价×数量；无运费/税。

### TC-015 checkout 失效兜底｜ API-014
- 取不存在 checkoutId → 4004；前端显示 `会话已失效`(COPY-034)/`二维码已失效`(COPY-035)。

### TC-016 重复支付｜ API-015
- 同一 checkout 二次 pay → 4009。

### TC-022 上下架机制｜ REQ-035
1. Admin 下架 phy_car_001。
2. 前台首页/分类/搜索均不展示 phy_car_001；详情入口不可达。
3. 若购物车已有该商品，前往结算返回 4009 并提示 COPY-045。
4. 已生成历史订单详情仍可查看。
- **断言**：下架影响可见性与新 checkout，不影响历史订单。

### TC-023 售罄态｜ REQ-042
1. Admin 将 phy_ele_001 stock 改为 SOLD_OUT。
2. 前台首页/分类/搜索仍展示该商品并显示 `已售罄`(COPY-046)。
3. 详情页购买/加购按钮置灰。
4. 强制加购或 checkout 返回 4009，前端提示 COPY-046。
- **断言**：售罄不同于下架；仍展示但不可买。

### TC-024 Admin 商品/Banner/服务 CRUD｜ REQ-037/038/039
1. Admin 新增/编辑商品字段并保存。
2. 前台重新进入首页/分类/详情，展示更新后的内容。
3. Admin 编辑 Banner target 与排序，前台首页下一次读取生效。
4. Admin 编辑展示服务说明，P5 下一次读取生效。
- **断言**：字段校验失败返回 4000；target 不存在不保存；前台不需重新构建即可看到已保存改动。

### TC-025 Admin 订单与会话｜ REQ-040/041
1. 完成一张实物订单和一张会员订单。
2. Admin 订单列表按类型/状态筛选，进入详情。
3. Admin session 展示 auth/drive/net/cartCount/checkout/orderCount/membership。
4. 可选 Admin reset 后前台订单/购物车/会员清空，商品/Banner/服务改动保留。
- **断言**：Admin 订单只读；session reset 作用域与 PRD §15.10.7 一致。
