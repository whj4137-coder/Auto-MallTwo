## 背景
实物购物闭环是 Demo 的主线，用来演示"驻车低干扰购物"的完整路径（首页 → 详情 → 购物车 → 确认 → 模拟支付 → 订单）。同时演示展示服务（充电/生活）不可购买的边界。

## 定义范围（做什么）
- 浏览：首页推荐、类目筛选、搜索（商品名 substring）。
- 实物详情：颜色/容量 SKU（≤2 层），数量 [1,5]，价格=单价×数量。
- 加入购物车（仅 PHYSICAL，默认数量 1）；**同一 SKU 重复加购合并数量**（受 [1,5] 上限约束），不同 SKU 独立行；购物车勾选/改量/删除（无二次确认）/多选结算。
- 立即购买：独立临时 checkout（source=BUY_NOW），**不影响购物车**。
- 确认订单 → QR 模拟支付（模拟扫码成功）→ 写订单（ORDER-P-NNN，状态「已支付」）→ 支付结果 → 订单中心。
- 展示服务详情仅「暂未开放」，强制写入返回 2003。
- 金额以分（priceCents）计算，显示 `¥` 整数。

## 明确排除（不做什么）
- 不做真实支付 / 真实扣款。
- 不做运费、优惠券、税、积分。
- 不做商品评价、推荐算法、库存扣减。
- 会员开通不在本 spec（见 SPEC-002）。
- 门禁/登录/重置的状态机不在本 spec（见 SPEC-003，本 spec 仅遵循其前置）。

## 验收标准
- [ ] 首页 HERO Banner → phy_car_001 详情（REQ-001）。
- [ ] 搜索「磁吸」命中 phy_car_001；「不存在的词」→ 空态 + 返回推荐（REQ-003）。
- [ ] phy_ele_001 选 128G、数量 2 → 价格 ¥998（REQ-004）。
- [ ] 加购成功 toast「已加入购物车」，购物车新增对应 SKU 行（REQ-007）。
- [ ] 购物车勾选项结算 → 确认订单（¥129）→ 模拟扫码成功 → ORDER-P-001 / 已支付，已勾选项被删（A-01）。
- [ ] 立即购买全程购物车行数/数量/勾选不变（REQ-008 / A-02）。
- [ ] 展示服务详情无购买入口；强制 `POST /cart(svc_charge_001)` → 2003（REQ-006 / C-01）。
- [ ] 订单中心时间降序、不分页；点订单卡进入 P13 订单详情（REQ-014/034）。
- [ ] 订单详情只读，实物显示收货信息、会员显示权益/绑定车辆，返回回订单中心（REQ-034）。
- [ ] 所有金额整数显示、计算无运费/税（§10.6）。
