## 背景
演示者通过 Demo Bar 手动控制登录/驾驶/网络三组状态，驱动门禁行为；这是"行车/断网安全门禁"与"未登录续作"演示的核心。重置用于一键回到干净演示态。

## 定义范围（做什么）
- Demo Bar：三组状态开关（GUEST/LOGGED_IN、PARKED/DRIVING、ONLINE/OFFLINE），手动切换、不自动触发、active 高亮、常驻显示组合状态文字；Demo Bar 自身按钮不被门禁拦截。
- 全局门禁：写入须 `LOGGED_IN AND PARKED AND ONLINE`；优先级 `DRIVING > OFFLINE > GUEST`，只显示最高优先级提示。
- 未登录续作：GUEST 点交易 → LoginDialog（预填 admin/123456）→ 成功关闭并自动续作；密码错保留并提示；点窗外不关闭。
- 行车拦截：交易按钮置灰 + toast；确认/QR 页切行车主按钮即时置灰，切回恢复；输入类禁用 + 语音占位。
- 断网拦截：交易按钮置灰 + toast；已加载可浏览；购物车/订单中心只读可进。
- 重置：无二次确认，清 cart/checkout/orders/memberships + 前端 TokenStore/session，seqP/seqM 归零，恢复默认并跳首页。
- toast 规则：同时刻仅一条、按优先级取最高、不堆叠。

## 明确排除（不做什么）
- 不接车辆真实信号/传感器（drive/net 纯手动）。
- 不接真实鉴权后端（登录态仅内存，重启回 GUEST）。
- 不做真实语音（语音占位仅提示文案）。
- 不做重置的二次确认弹窗。

## 验收标准
- [ ] 三组状态仅 Demo Bar 可切，刷新/重启回默认 GUEST+PARKED+ONLINE（REQ-016/017/018）。
- [ ] GUEST 点 加购/立即购买/开通会员/进购物车/进订单中心 → LoginDialog；正确密码续作，错误密码显「账号或密码不正确」（REQ-020 / AUTH-01）。
- [ ] 行车下交易按钮置灰，点击 toast「行车中仅支持浏览，请在驻车后继续操作」；确认/QR 页切行车主按钮置灰、切回恢复（REQ-021 / D-01）。
- [ ] 断网下置灰 + toast「当前网络不可用，请检查网络后重试」；可只读进购物车/订单（REQ-022 / D-02）。
- [ ] 行车+断网+未登录并存时只显示行车提示，toast 不堆叠（REQ-024/029）。
- [ ] 重置无弹窗、全清、状态复位、跳首页、下一实物单号 ORDER-P-001（REQ-019 / DEMO-01）。
- [ ] Demo Bar 常驻显示组合状态文字且实时更新（REQ-032）。
- [ ] 错误码 1001/1002/2001/2002/2003 → 对应前端行为（REQ-025 / §12）。
