## 背景
会员服务与实物购物的差异化是核心演示点：会员**不进购物车**，直接「立即开通」，支付后置 membership=ACTIVE 并改变详情按钮态与「我的」页展示。

## 定义范围（做什么）
- 会员详情：价格、权益清单、有效期、默认绑定车辆（特斯拉 Model Y）。
- 「立即开通」→ 确认开通页（标题「确认开通」，不进购物车）→ QR 模拟支付。
- 支付成功：生成 ORDER-M-NNN（状态「已开通」）+ membership=ACTIVE。
- 已开通后：详情主按钮变「已开通 · 查看订单」（跳订单中心）。
- 「我的」会员区块实时显示「已开通 / 未开通」。
- **互斥 + 幂等**：membership 为全局单一状态，月卡/年卡互斥——开通任一即 ACTIVE，此后两张会员详情均显示「已开通 · 查看订单」，不重复下单。

## 明确排除（不做什么）
- 不做真实支付。
- 不做会员到期/续费/退订（仅展示有效期数值）。
- 不做权益的真实兑换或核销。
- 会员订单不可加入购物车、不参与多选结算。

## 验收标准
- [ ] mem_001 详情显示 ¥39 + 权益（专属主题/车主精选权益/会员日特惠）+ 30 天 + 绑定 Model Y（REQ-005）。
- [ ] 确认页标题为「确认开通」（REQ-009）。
- [ ] 模拟扫码成功 → ORDER-M-001 / 已开通 + membership=ACTIVE（B-01）。
- [ ] 支付结果标题「权益已开通」（REQ-013 协同）。
- [ ] 回 mem_001 详情按钮变「已开通 · 查看订单」（REQ-005）。
- [ ] 「我的」会员区块显示「已开通」（REQ-015）。
- [ ] 已 ACTIVE 再进开通流不产生第二张会员订单（REQ-033）。
- [ ] 订单号 ORDER-M 独立于 ORDER-P 递增，重置后回 001（§10.5）。
