## 背景
系统并非单纯前端 Mock：第一版包含可运营的 Admin 管理后台，且数据为后端种子、Admin 改、前台实时读。上下架是贯穿商品/Banner/服务的可见性机制。

## 定义范围（做什么）
- **后台登录**（REQ-036）：固定账号（可共用 admin/123456）。
- **商品管理**（REQ-037）：实物+会员商品 列表/新增/编辑/上下架/分类/价格/图片/详情/SKU/库存(模拟)/排序。
- **Banner 管理**（REQ-038）：CRUD/上下架/跳转目标/顺序。
- **服务内容管理**（REQ-039）：会员权益·价格·上下架·详情；展示服务介绍·暂未开放说明·后续能力说明·上下架。
- **订单管理**（REQ-040）：列表/详情/类型·状态筛选/用户·车辆·金额·时间/模拟支付状态（只读查询）。
- **演示会话管理**（REQ-041）：展示运行快照（购物车数/checkout/订单数/会员态/序号）；登录·驾驶·网络为各客户端 Demo 态、服务端不持有；可选后台重置。
- **账号信息**（change 0006，本次冻结纳入）：只读展示个人信息 + 车辆信息（源自 §10.4 固定 Mock），本期不可编辑。
- **上下架机制**（REQ-035）：`published` 控制前台可见性；下架不在首页/分类/搜索出现；购物车含下架品结算拦截（4009 → COPY-045）；历史订单不受影响；前台只读接口仅返回 published=true。
- 数据为后端种子，Admin 改动经后端持久化、前台实时读取。

## 明确排除（不做什么）
- 发货 / 退款 / 售后 / 改价后重新支付 / 关闭订单等真实电商运营。
- 真实库存扣减（库存为展示/模拟字段）。
- Admin 不接真实支付/物流。

## 验收标准
- [ ] 后台六模块可登录访问（REQ-036）。
- [ ] 商品/Banner/服务的新增·编辑·上下架·排序在前台**实时生效**（REQ-037/038/039）。
- [ ] 下架商品不在首页/分类/搜索出现；其详情入口不可达（REQ-035）。
- [ ] 购物车含下架商品时结算被拦并提示 COPY-045；历史订单不受影响（REQ-035）。
- [ ] 订单管理类型/状态筛选可用，字段齐全，只读（REQ-040）。
- [ ] 演示会话实时反映前台状态；可选重置生效（REQ-041）。
